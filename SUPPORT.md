# Support

This document explains how to get help with Specter-Tree, suggest improvements, or start a commercial conversation.

## Technical Support

Use GitHub Issues for bugs, broken setup flows, incorrect query results, or documentation problems.

When opening an issue, include:

- what you expected to happen
- what actually happened
- reproduction steps
- your OS, Bun version, and project framework when relevant
- logs or screenshots if they help explain the problem

## Feature Requests

Use GitHub Issues to request:

- new MCP tools
- new framework resolvers
- TypeScript analysis improvements
- documentation or onboarding improvements

Be specific about the workflow problem you are trying to solve. Requests framed around a concrete debugging or navigation task are easier to evaluate.

## Commercial Support

Need Specter-Tree adapted for a custom stack or larger AI-agent workflow?

The maintainer can discuss:

- custom MCP integrations
- TypeScript architecture guidance
- AI-assisted development workflow design

Open an issue describing your use case, or contact the maintainer directly if you already have an established channel.
